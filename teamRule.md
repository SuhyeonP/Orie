### commit message<br/>
* fix : 기능 (버그수정)
* feat : 추가기능
* chore : 코드적인 부분을 제외하고, eslint파일이나 gitignore같은 파일들을 수정 추가 제거등등을 할때 
* docs : 리드미파일
* refac : 폴더 추가, 수정, 제거 등등
* style : 쓰긴하지만 fix를 더 많이 쓴다고 한다. 
* test : unit test
* git commit -am "fix: blah blah" 마지막에 `.`찍기 노노
* yarn 을 사용한다. npm 놉!
* 브런치 이름은 소문자와, _로 사용,.,,카멜 노노...
